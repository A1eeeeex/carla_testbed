Purpose: hold algorithm stacks (Autoware), adapters, and plugin nodes that consume the IO contract.

Structure
- `baselines/autoware/`: docker compose + interface configs for Autoware direct-to-CARLA mode.
- `nodes/control/carla_control_bridge/`: ROS2 node bridging `AckermannDriveStamped` -> CARLA control for Mode-1.
- `plugins/`: placeholders for perception/planning/E2E ROS2 packages.

Quick start
- Mode-1 bridge: `python algo/nodes/control/carla_control_bridge/ros2_autoware_to_carla.py --control-topic /tb/ego/control_cmd`
- Mode-2 Autoware: `python io/scripts/run.py --profile io/contract/profiles/autoware_direct.yaml`

Notes
- Plugin dirs are intentionally empty; add your ROS2 packages or adapters per slot/topic contract.
- Autoware config files are generated by `io/backends/autoware_direct.py` each run.
